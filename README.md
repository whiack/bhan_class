# 教务管理综合平台

## 文件说明

- **index.html** — 完整前端应用，双击或部署到服务器即可使用
- **data.json** — 默认数据备份（学生总览 + 课表 + KPI），发给 AI 改完后再合并回 HTML

## 分享/部署成链接

这是纯静态网页，部署时只需要上传整个 `教务平台` 文件夹，入口文件是 `index.html`。

推荐方式：

1. **GitHub Pages / Netlify / Vercel / Cloudflare Pages**：上传 `index.html` 和 `data.json`，发布后会得到一个 `https://...` 链接，别人点开即可使用。
2. **内网或本机临时预览**：在本文件夹运行静态服务器后访问 `http://127.0.0.1:4173/`，只适合自己电脑预览，不适合发给外部人员。
3. **直接发送 HTML 文件**：对方下载后双击可以打开，但这不是公网链接。

注意：网页数据默认保存在每个人自己浏览器的 `localStorage` 中。也就是说，公开链接适合“每个人各自维护一份数据”。如果需要多人共用同一份实时数据，需要再接一个在线数据库或后端服务。

在线链接里的推荐数据流：

1. 点「📦 下载备份」导出完整 JSON。
2. 换电脑/换浏览器/发给别人时，点「导入备份」恢复数据。
3. 「绑定JSON / 新建绑定」更适合 Chrome/Edge 本地文件管理场景，公网链接里不一定可用。

## 数据修改流程

1. 把 `data.json` 丢给 AI，告诉它要改什么
2. 改完后的 JSON 发回给我，我帮你合并进 `index.html` 并重新部署
3. 或者直接在网页上点「📦 备份数据」导出你当前修改后的 JSON

## 数据结构

```
{
  students: [...],   // 学生总览（38人）
  kpiData: {...},    // KPI看板（4个季度，目标21/季）
  schedule: {...}    // 课表管理（7时段 × 5天，周日-周四）
}
```

### 学生字段

```
{
  id: 数字,
  name: "姓名",
  group: "年级",
  coming: true/false,
  master: true/false,
  vip: "VIP等级",
  direction: "申请方向",
  completed: false,
  courses: {
    "基础课1": { status: "none|doing|done|other", text: "项目名" },
    "基础课2": ...,
    "桌游项目": ...,
    "Unity项目": ...,
    "项目3": ...,
    "项目4": ...
  }
}
```

### KPI 字段

```
{
  name: "学生名",
  project: "项目名",
```
